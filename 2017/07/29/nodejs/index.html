node.js的模块化

  • process.argv （参数1：node.js的安装路径，参数2：执行文件的路径,参数3,4,5,6,7,8……：接收的命令行数据……）

    模块化开发的优点

  • 前端标准模块化规范(一般是异步)
    • AMD require.js(国外)
    • CMD sea.js(阿里)

  • 服务器的模块化规范(一般是同步的)

    • CommonJs Node.js

  • 模块化的规则

    • 一个js文件就是一个模块，模块内部的成员是相互独立的
    • 模块成员的导出和引入
    • 模块的导出和引入

      • exports（导出） require（引入）

    • 导出方式1(导出的是一个对象)

//文件1.js(导出)
var sum = function(a,b){
return a + b;
}
module.exports= sum;
//文件2.js(引入)
var sum = require(‘./文件1.js’);
var ret = sum(1,2)
